Create and share javascript unit tests

Name Your Test

http://assertion.me/

Everything starts with a name. Pick a good one that's recognizable and descriptive.

Examples of names include:

  • creating an Object.assign polyfill
  • setTimeout craziness
  • function composition methods

Any name will do, although it must be unique. How else are you going to persist these assertions without signing up for yet another service?

Assert (Expect) Something

'use strict';

const expect = chai.expect;
const assert = chai.assert;

// lets write some tests for the implementation
// of the function 'sum()' to be sure it's
// behaving properly

let even = sum(2, 4, 6, 8);
let odds = sum(0, 1, 3, 5, 7);
let arr = sum([ 5, 5, 5 ]);
let obj = sum({ 'a': 10, 'b': 20, 'c': 30 });

expect(sum).to.be.a('function');
expect(arr).to.equal(15);
expect(obj).to.equal(60);
assert.strictEqual(even, 20);
assert.strictEqual(odds, 16);

The test code and the assertion code (what you are able to write now) will be executed in the same sandbox. Therefore, any functions defined in the tests are the functions to implement and be tested.

The testing sandbox comes with the Chai Assertion Library as a global variable and you can code using most ES6 features.

Some sample tests are provided because sometimes examples are easier to understand than reading.

Oh, and don't worry if you can't think up all of the tests now. You'll be able to change them whenever you want.
